# نیمروز دسکتاپ

**نیمروز** — دستیار هوش مصنوعی متن‌باز و فارسی برای دسکتاپ، با تمرکز روی **فضای کاری عامل‌محور** (Agentic Workspace). ساخته‌شده با Electron، React و [Vercel AI SDK](https://github.com/vercel/ai).

[English](README.md)

> **نسخه ۱٫۰٫۰** — نیمروز دیگر فقط یک چت‌بات ساده نیست. یک فضای کاری عامل‌محور است: گفتگو به‌همراه ابزارهایی که می‌توانند داخل پوشهٔ پروژه‌تان کار کنند، با تأییدیه، آرتیفکت و ذخیره‌سازی محلی.

> **نرم‌افزار آزمایشی** — ممکن است باگ یا گوشه‌های ناتمام داشته باشد. لطفاً تست کنید و [مشکلات را در گیت‌هاب گزارش دهید](https://github.com/xmannii/nimruz-desktop/issues).

با حساب ChatGPT به Codex، به [OpenRouter](https://openrouter.ai/) یا هر ارائه‌دهندهٔ سازگار با OpenAI وصل شوید. مدل‌های ابزارمحور می‌توانند داخل پوشهٔ فضای کاری با تأیید شما فایل بخوانند، جستجو کنند، دستور اجرا کنند و آرتیفکت بسازند؛ Codex به‌صورت گفتگویی و در یک محیط ایزوله اجرا می‌شود. همهٔ داده‌های نیمروز روی دستگاه خودتان می‌ماند.

### نصب در macOS

1. آخرین `.dmg` را از [Releases](https://github.com/xmannii/nimruz-desktop/releases/latest) بگیرید.
2. DMG را باز کنید و **Nimruz** را به **Applications** بکشید.
3. برنامه را با یکی از روش‌های زیر باز کنید.

بیلدهای مک هنوز code-sign / notarize نشده‌اند؛ Gatekeeper ممکن است برنامه را مسدود کند. پیام **«Nimruz is damaged…»** معمولاً گمراه‌کننده است — فایل خراب نیست؛ مک دانلود بدون امضا را رد می‌کند.

**پیشنهادی:** روی Nimruz در Applications راست‌کلیک → **Open** → دوباره **Open**. فقط یک‌بار لازم است.

**جایگزین:** پرچم quarantine را بردارید، بعد عادی باز کنید:

```bash
xattr -dr com.apple.quarantine /Applications/Nimruz.app
```

**اگر قبلاً مسدود شده:** **System Settings → Privacy & Security** و کنار Nimruz گزینهٔ **Open Anyway**.

### نصب در ویندوز

1. آخرین `.exe` را از [Releases](https://github.com/xmannii/nimruz-desktop/releases/latest) بگیرید.
2. نصب‌کننده را اجرا کنید و مراحل را دنبال کنید.

## امکانات

### فضای کاری عامل‌محور

- **فضاهای کاری** — اتصال پوشهٔ پروژه؛ گفتگوها زیر هر فضای کاری (به‌همراه فضای کاری خانه)
- **ابزارهای عامل** — خواندن/فهرست/جستجوی فایل، نوشتن و پچ، شِل، آرتیفکت و تسک
- **تأییدیه‌ها** — ابزارهای پرریسک قبل از اجرا سؤال می‌کنند؛ امکان «همیشه مجاز» برای هر فضای کاری
- **پنل کناری** — فایل‌ها، آرتیفکت‌ها، تسک‌ها، فعالیت و تنظیمات فضای کاری
- **زمینه در کامپوزر** — الصاق فایل/آرتیفکت، انتخاب فضای کاری و `@`-منشن

### گفتگو و مدل‌ها

- **چت استریم** — مارک‌داون، کد، ریاضی (KaTeX)، Mermaid، رابط راست‌به‌چپ فارسی
- **تایم‌لاین ابزار** — استدلال و فراخوانی ابزارها به‌صورت متصل؛ اجراهای طولانی در یک خلاصهٔ جمع‌وجور
- **Codex با ورود ChatGPT** — همگام‌سازی مدل‌های مجاز حساب و اجرای گفتگو در محیط محدود و ایزوله
- **OpenRouter و ارائه‌دهندهٔ سفارشی** — مرور مدل‌ها، پیش‌فرض، شدت استدلال اختیاری
- **خواندن وب** — ابزار `fetch_url` برای صفحات عمومی (امن، HTML به متن)
- **عنوان خودکار** — نام گفتگو از اولین پیام
- **مدیریت گفتگو** — سنجاق، خروجی Markdown/JSON، کپی/بازتولید، جستجوی تاریخچه

### شخصی‌سازی و مهارت‌ها

- **حافظه** — نکات ماندگار که دستیار می‌تواند ذخیره یا فراموش کند
- **متخصص‌ها** — متخصص‌های قابل‌استفاده مجدد با `/` در چت
- **مهارت‌ها** — بسته‌های `SKILL.md` که در صورت نیاز با `load_skill` بارگذاری می‌شوند
- **شخصی‌سازی** — سبک پاسخ، دستورالعمل‌های سفارشی، زمینهٔ پروفایل

### ظاهر و تجربه

- **ظاهر** — روشن/تیره/سیستم، پالت رنگ (از جمله تم نیمروز)، انتخاب فونت سیستم
- **راهنمای شروع** — تور اولین اجرا (ظاهر، مدل‌ها، فضاهای کاری، چت)
- **محلی‌محور** — SQLite در `userData`؛ کلید API در keychain سیستم‌عامل
- **انتشار خودکار** — با تغییر نسخه روی `main`، نصب‌کنندهٔ ویندوز و مک ساخته می‌شود

## شروع سریع (توسعه)

### پیش‌نیازها

- [Node.js](https://nodejs.org/) نسخهٔ ۲۲٫۱۲٫۰ یا جدیدتر
- [pnpm](https://pnpm.io/) نسخهٔ ۹ یا جدیدتر

### نصب و اجرا

```bash
git clone https://github.com/xmannii/nimruz-desktop.git
cd nimruz-desktop
pnpm install
pnpm dev
```

در اولین اجرا:

1. تور کوتاه شروع را ببینید (یا رد کنید).
2. از **تنظیمات → مدل‌ها** حساب ChatGPT را برای Codex وصل کنید، کلید OpenRouter را وارد کنید، یا ارائه‌دهندهٔ دیگری بسازید.
3. یک **فضای کاری** بسازید/باز کنید و در صورت نیاز پوشهٔ پروژه را وصل کنید.

کلید از طریق Keychain مک، DPAPI ویندوز، یا libsecret/KWallet لینوکس رمزنگاری می‌شود. روی لینوکس اگر فقط backend ناامن `basic_text` در دسترس باشد، ذخیره رد می‌شود.

برای استفادهٔ عادی به فایل `.env` نیاز نیست — اعتبارنامه‌ها داخل برنامه مدیریت می‌شوند.

### استفاده از Codex با حساب ChatGPT

1. به **تنظیمات → مدل‌ها → Codex** بروید و ورود با ChatGPT را شروع کنید.
2. ورود OpenAI را در مرورگر کامل کنید؛ ورود با کد دستگاه نیز در دسترس است.
3. مدل‌های Codex حساب را همگام کنید و یکی را برای گفتگو انتخاب کنید.

Codex در نیمروز به پوشه‌های متصل، شِل، ابزارهای وب، آرتیفکت‌ها یا تسک‌های فضای کاری دسترسی ندارد و نباید ادعا کند این عملیات را انجام داده است. برای اجرای عامل روی فایل‌های واقعی از یک مدل ابزارمحور OpenRouter یا ارائه‌دهندهٔ سازگار استفاده کنید.

### ساخت نصب‌کننده

```bash
pnpm dist
```

خروجی در پوشهٔ `release/` قرار می‌گیرد.

## معماری (خلاصه)

```
Electron main (Node)
├─ سرور محلی احرازهویت‌شده
│  ├─ POST /api/chat   → عامل فضای کاری یا Codex app-server
│  ├─ POST /api/agent/run → عامل فضای کاری (فایل، شِل، آرتیفکت، تسک)
│  └─ POST /api/chat/title → عنوان خودکار
├─ SQLite      → گفتگوها، فضاهای کاری، آرتیفکت، تسک، اجراها، حافظه، متخصص‌ها، نگاشت رشته‌های Codex
├─ فایل‌های فضای کاری → مسیرهای محدود به ریشهٔ متصل‌شده
├─ مهارت‌ها    → ~/.nimruz/skills
├─ Codex app-server → ورود ChatGPT، مدل‌ها و رشته‌های ایزوله
└─ پنجرهٔ رندر → Vite / React (سندباکس)
```
